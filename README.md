# Sistema-de-Academia
Este é o projeto final da matéria de Engenharia de Software da UDESC CCT 2025/2.

# Processo de desenvolvimento
Na fase 1 do projeto foi estabelecido o problema que o sistema procura resolver, o escopo, a estimativa de duração do projeto e o diagrama UML.
Na fase 2 o projeto foi desenvolvido desta forma e nesta ordem, utilizando o VSCode com extensão para java:
  1º Tradução do diagrama para um projeto em java maven, apenas o esqueleto do código foi feito sem implementações dos métodos.
  2º O desenvolvimento dos testes unitários das classes mais críticas para o sistema, elas foram escolhidas por serem o mínimo necessário para uma academia funcionar.
  3º Após a criação dos testes com JUnit, foi feita a implementação necessária dos métodos nas classes para que os testes funcionem, também foram adicionados setters e getters.
Os testes cobrem apenas as classes e métodos críticas para o sistema.

#Execução
Abra a pasta academia no VSCode, se as extensões _Extension Pack for Java_ e _Test Runner for Java_ estiverem instaladas, deve ser possível rodas os testes unitários.
